# 问题反馈指南

在提交 Issue 前，请务必阅读以下内容，这将极大提高问题解决的效率。

## 1. 核心原则

> [!IMPORTANT]
> **请先更新到最新版本**！许多问题可能已在最新版中修复。

## 2. 收集必要信息 (必填)

提交 Bug Report 时，**必须**提供以下信息：

### 📂 日志文件 (Logs)

- **位置**：`debug` 文件夹。
  - *提示：点击"运行日志"面板右上角的 📂 图标可直接打开。*
- **操作**：请将**整个 `debug` 文件夹**打包为 `.zip` 或 `.7z` 上传。
  - ❌ 不要复制文字发送。
  - ❌ 不要截图日志。
  - ❌ 不要只上传单个文件。

### 🖼️ 关键截图

1. **软件画面**：问题发生时的完整软件界面截图。
2. **版本信息**：`设置` -> `调试` 页面中的版本信息截图。

## 3. 特殊情况 (选填)

### 👁️ 识别问题 (漏点/点错)

1. 在设置中开启 **"保存调试图像"**。
2. 再次尝试复现问题。
3. 上传 `debug/vision/` 目录下生成的调试图片。
4. 若文件过大无法上传，可上传到 QQ 群文件 (1078075580)，并在 issue 里备注具体文件名。
5. 反馈完成记得关闭，不然硬盘会爆炸（确信

### 💥 程序崩溃 (闪退)

若程序直接消失或弹出错误框，请尝试提供 Crash Dump 文件：

- **路径**：`C:\Users\<用户名>\AppData\Local\CrashDumps\`
- **文件**：最近生成的 `.dmp` 文件。
